PHASE 5 — ADF Self-hosted Integration Runtime (Manual — UI only)
──────────────────────────────────────────────────────────────────────────────
Purpose : Bridge between your local SQL Server and ADF in the cloud.
          Without this, ADF cannot reach on-premises data sources.

Steps:
  1. Open ADF Studio → https://adf.azure.com
     Select factory: adf-data-260524-dev

  2. Left menu → Manage → Integration Runtimes → + New

  3. Select: Azure, Self-Hosted → Continue
     Select: Self-Hosted → Continue
     The Self-hosted IR works outbound only:
     Your machine ──── outbound HTTPS (port 443) ────► ADF in Azure

  4. Name        : ir-selfhosted-dev
     Description : Self-hosted IR for on-prem SQL Server (AdventureWorksLT2019)
     → Click Create

  5. On the next screen, copy Authentication Key 1 (keep this — needed in step 7)

  6. Click "Download and install" → download the MSI installer
     Run the installer on YOUR LOCAL WINDOWS MACHINE
     (same machine where SQL Server is installed)

  7. During setup, paste Authentication Key 1 when prompted → Register

  8. Back in ADF Studio → Integration Runtimes
     Confirm ir-selfhosted-dev shows status: Running ✅

Notes:
  - IR must remain running whenever ADF pipelines execute
  - If your machine restarts, the IR service auto-starts (installed as Windows service)
  - To verify: Windows Services → "Integration Runtime Service" → Running

─── 6.6 Pipeline: pl-ingestion-sqlserver-to-bronze ──────────────────────────

  1. ADF Studio → Author → Pipelines → + New pipeline
     Name: pl-ingestion-sqlserver-to-bronze

  2. Add activity: Lookup
       Name         : lkp-get-all-tables
       Source       : ls-sqlserver-onprem (inline query)
       Query        :
         SELECT TABLE_SCHEMA, TABLE_NAME
         FROM INFORMATION_SCHEMA.TABLES
         WHERE TABLE_SCHEMA = 'SalesLT'
           AND TABLE_TYPE  = 'BASE TABLE'
       First row only: OFF

  3. Add activity: ForEach (connect from Lookup success)
       Name         : foreach-tables
       Items        : @activity('lkp-get-all-tables').output.value
       Batch count  : 4  (parallel copies — safe for dev)

  4. Inside ForEach → Add activity: Copy data
       Name         : copy-table-to-bronze
       Source:
         Dataset    : ds_sqlserver_source
         schemaName : @item().TABLE_SCHEMA
         tableName  : @item().TABLE_NAME
         Query      : SELECT * FROM [@{item().TABLE_SCHEMA}].[@{item().TABLE_NAME}]
       Sink:
         Dataset    : ds_adls_bronze_parquet
         schemaName : @item().TABLE_SCHEMA
         tableName  : @item().TABLE_NAME

  5. Publish All

  6. Debug run → Monitor tab → should show all SalesLT tables copied to bronze ✅
     Verify in Portal: sadataeng260524dev → bronze → SalesLT → [TableName]/

─── 7.2 Upload Notebooks ────────────────────────────────────────────────────

  1. Databricks → Workspace → your username folder → Create folder: notebooks

  2. Upload or create 3 notebooks:

     ┌─────────────────────────────────────────────────────────────────────┐
     │ Notebook 1: storagemount                                            │
     │ Purpose   : Mount ADLS Gen2 containers using Service Principal OAuth│
     └─────────────────────────────────────────────────────────────────────┘

     # Cell 1 — config (read SP credentials from Databricks Secret Scope)
     configs = {
       "fs.azure.account.auth.type"                          : "OAuth",
       "fs.azure.account.oauth.provider.type"               : "org.apache.hadoop.fs.azurebfs.oauth2.ClientCredsTokenProvider",
       "fs.azure.account.oauth2.client.id"                  : dbutils.secrets.get("kv-scope", "sp-client-id"),
       "fs.azure.account.oauth2.client.secret"              : dbutils.secrets.get("kv-scope", "sp-client-secret"),
       "fs.azure.account.oauth2.client.endpoint"            : f"https://login.microsoftonline.com/{dbutils.secrets.get('kv-scope','sp-tenant-id')}/oauth2/token"
     }

     # Cell 2 — mount all 3 containers
     STORAGE_ACCOUNT = "sadataeng260524dev"

     for container in ["bronze", "silver", "gold"]:
         mount_point = f"/mnt/{container}"
         if not any(mount.mountPoint == mount_point for mount in dbutils.fs.mounts()):
             dbutils.fs.mount(
                 source       = f"abfss://{container}@{STORAGE_ACCOUNT}.dfs.core.windows.net/",
                 mount_point  = mount_point,
                 extra_configs = configs
             )
             print(f"✅ Mounted: {mount_point}")
         else:
             print(f"⏭️  Already mounted: {mount_point}")

     # Cell 3 — verify
     display(dbutils.fs.mounts())

     ┌─────────────────────────────────────────────────────────────────────┐
     │ Notebook 2: bronze_to_silver                                        │
     │ Purpose   : Convert datetime columns → date, write Delta to silver  │
     └─────────────────────────────────────────────────────────────────────┘

     from pyspark.sql import functions as F
     from pyspark.sql.types import DateType
     import os

     BRONZE_PATH = "/mnt/bronze/SalesLT"
     SILVER_PATH = "/mnt/silver/SalesLT"

     tables = [t.name for t in dbutils.fs.ls(BRONZE_PATH)]

     for table in tables:
         df = spark.read.parquet(f"{BRONZE_PATH}/{table}/{table}.parquet")

         # Convert all TimestampType / DateType columns to DateType
         for col_name, col_type in df.dtypes:
             if col_type in ("timestamp", "date"):
                 df = df.withColumn(col_name, F.col(col_name).cast(DateType()))

         df.write.format("delta").mode("overwrite").save(f"{SILVER_PATH}/{table}")
         print(f"✅ {table} → silver")

     ┌─────────────────────────────────────────────────────────────────────┐
     │ Notebook 3: silver_to_gold                                          │
     │ Purpose   : Rename columns PascalCase → UPPER_SNAKE_CASE, write     │
     │             Delta to gold                                            │
     └─────────────────────────────────────────────────────────────────────┘

     import re

     SILVER_PATH = "/mnt/silver/SalesLT"
     GOLD_PATH   = "/mnt/gold/SalesLT"

     def to_upper_snake(name):
         s = re.sub(r'([A-Z])', r'_\1', name).lstrip('_')
         return s.upper()

     tables = [t.name for t in dbutils.fs.ls(SILVER_PATH)]

     for table in tables:
         df = spark.read.format("delta").load(f"{SILVER_PATH}/{table}")

         renamed = {c: to_upper_snake(c) for c in df.columns}
         for old, new in renamed.items():
             df = df.withColumnRenamed(old, new)

         df.write.format("delta").mode("overwrite").save(f"{GOLD_PATH}/{table}")
         print(f"✅ {table} → gold")

PHASE 10 — Synapse Serverless SQL Views
──────────────────────────────────────────────────────────────────────────────
Purpose : Expose the Gold Delta Lake files as SQL views in Synapse so
          Power BI can query them with standard SQL without a dedicated pool.

─── 10.1 Create Serverless Database ─────────────────────────────────────────

  1. Portal → synapse-dataeng-dev → Open Synapse Studio

  2. Left menu → Develop → + SQL script

  3. Run:
       IF NOT EXISTS (SELECT * FROM sys.databases WHERE name = 'gold_db')
           CREATE DATABASE gold_db;

─── 10.2 Grant Synapse Access to ADLS ───────────────────────────────────────

  Portal → sadataeng260524dev → Access Control (IAM) → Add role assignment
    Role     : Storage Blob Data Reader
    Assign to: Managed Identity → synapse-dataeng-dev

─── 10.3 Create Views for Each Gold Table ───────────────────────────────────

  In Synapse Studio → Develop → + SQL script → connect to gold_db

  Template for each table (repeat per SalesLT table):

    CREATE OR ALTER VIEW SalesLT_Customer AS
    SELECT *
    FROM OPENROWSET(
        BULK 'https://sadataeng260524dev.dfs.core.windows.net/gold/SalesLT/Customer/',
        FORMAT = 'DELTA'
    ) AS [result];

  Tables to create views for:
    - Customer
    - Product
    - ProductCategory
    - SalesOrderHeader
    - SalesOrderDetail
    - Address
    - CustomerAddress

─── 10.4 Create Stored Procedure (Dynamic — creates all views at once) ──────

    CREATE OR ALTER PROCEDURE sp_create_gold_views
    AS
    BEGIN
        DECLARE @sql NVARCHAR(MAX)
        DECLARE @tables TABLE (tbl NVARCHAR(100))

        INSERT INTO @tables VALUES
            ('Customer'), ('Product'), ('ProductCategory'),
            ('SalesOrderHeader'), ('SalesOrderDetail'),
            ('Address'), ('CustomerAddress')

        DECLARE @tbl NVARCHAR(100)
        DECLARE cur CURSOR FOR SELECT tbl FROM @tables
        OPEN cur
        FETCH NEXT FROM cur INTO @tbl
        WHILE @@FETCH_STATUS = 0
        BEGIN
            SET @sql = N'
            CREATE OR ALTER VIEW SalesLT_' + @tbl + N' AS
            SELECT * FROM OPENROWSET(
                BULK ''https://sadataeng260524dev.dfs.core.windows.net/gold/SalesLT/' + @tbl + N'/'',
                FORMAT = ''DELTA''
            ) AS [result]'
            EXEC sp_executesql @sql
            FETCH NEXT FROM cur INTO @tbl
        END
        CLOSE cur
        DEALLOCATE cur
    END

    -- Run it:
    EXEC sp_create_gold_views

  Verify:
    SELECT TOP 10 * FROM SalesLT_Customer;  -- should return rows ✅

─── 11.3 Gender Column (Derived) ────────────────────────────────────────────

  The Customer table has a TITLE column (Mr./Ms./Mrs./Miss/Dr.).
  Derive gender from it:

  Power BI → Transform data → Add column → Custom column:
    Column name: Gender
    Formula:
      if [TITLE] = "Mr." or [TITLE] = "Dr." then "Male"
      else if [TITLE] = "Ms." or [TITLE] = "Mrs." or [TITLE] = "Miss" then "Female"
      else "Unknown"

─── 11.4 Build Dashboard Visuals ────────────────────────────────────────────

  Visual 1 — Card: Total Products Sold
    Value: COUNT(SalesLT_SalesOrderDetail[ORDER_QTY])

  Visual 2 — Card: Total Revenue
    Value: SUM(SalesLT_SalesOrderDetail[LINE_TOTAL])
    Format: Currency

  Visual 3 — Donut Chart: Gender Split
    Legend : SalesLT_Customer[Gender]
    Values : COUNT(SalesLT_Customer[CUSTOMER_ID])

  Visual 4 — Bar Chart: Revenue by Product Category
    Axis  : SalesLT_ProductCategory[NAME]
    Values: SUM(SalesLT_SalesOrderDetail[LINE_TOTAL])

  Slicers:
    - SalesLT_SalesOrderHeader[ORDER_DATE]     (date range slicer)
    - SalesLT_ProductCategory[NAME]            (product category)
    - SalesLT_Customer[Gender]                 (gender)
