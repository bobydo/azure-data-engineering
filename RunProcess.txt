PHASE 5 — ADF Self-hosted Integration Runtime (Manual — UI only)
Stage   : 🔌 Connect local machine → ADF
──────────────────────────────────────────────────────────────────────────────
Purpose : Bridge between your local SQL Server and ADF in the cloud.
          Without this, ADF cannot reach on-premises data sources.

Steps:
  1. Open ADF Studio → https://adf.azure.com
     Select factory: adf-data-260524-dev

  2. Left menu → Manage → Integration Runtimes → + New

  3. Select: Azure, Self-Hosted → Continue
     Select: Self-Hosted → Continue
     The Self-hosted IR works outbound only:
     Your machine ──── outbound HTTPS (port 443) ────► ADF in Azure

  4. Name        : ir-selfhosted-dev
     Description : Self-hosted IR for on-prem SQL Server (AdventureWorksLT2019)
     → Click Create

  5. On the next screen, copy Authentication Key 1 (keep this — needed in step 7)

  6. Click "Download and install" → download the MSI installer
     Run the installer on YOUR LOCAL WINDOWS MACHINE
     (same machine where SQL Server is installed)

  7. During setup, paste Authentication Key 1 when prompted → Register

  8. Back in ADF Studio → Integration Runtimes
     Confirm ir-selfhosted-dev shows status: Running ✅

Notes:
  - IR must remain running whenever ADF pipelines execute
  - If your machine restarts, the IR service auto-starts (installed as Windows service)
  - To verify: Windows Services → "Integration Runtime Service" → Running

  Read secret values (Azure Cloud Shell or local CLI):
    az keyvault secret show --vault-name kv-dataengproj-dev --name sql-username --query value -o tsv
    az keyvault secret show --vault-name kv-dataengproj-dev --name sql-password --query value -o tsv
    az keyvault secret show --vault-name kv-dataengproj-dev --name sp-tenant-id --query value -o tsv
    az keyvault secret show --vault-name kv-dataengproj-dev --name sp-client-id --query value -o tsv

    → scroll down to "Secret value" → click eye icon 👁

  Verify adf_user exists on your local SQL Server (run in SSMS):

  If it doesn't exist yet, create it:

  (login at server level ≠ user at database level):

    CREATE USER adf_user FOR LOGIN adf_user;
    ALTER ROLE db_datareader ADD MEMBER adf_user;

─── 6.5 Pipeline: pl-ingestion-sqlserver-to-bronze ──────────────────────────

  1. ADF Studio → Author → Pipelines → + New pipeline
     Name: pl-ingestion-sqlserver-to-bronze

  2. Add activity: Lookup
       Name         : lkpgetalltables 
       Note: integration running point to local\SQLExpress, 
       so following query still verify at 
       Source       : lssqlserveronprem (inline query)
       Query        :
         SELECT TABLE_SCHEMA, TABLE_NAME
         FROM INFORMATION_SCHEMA.TABLES
         WHERE TABLE_SCHEMA = 'Sales'
           AND TABLE_TYPE  = 'BASE TABLE'
       First row only: OFF

  3. Add activity: ForEach (connect from Lookup success)
       Name         : foreachtables
       Items        : @activity('lkpgetalltables').output.value
       Batch count  : 4  (parallel copies — safe for dev)

  4. Inside ForEach → Add activity: Copy data
       Name         : copytabletobronze
       Source:
         Dataset    : ds_sqlserver_source
         schemaName : @item().TABLE_SCHEMA
         tableName  : @item().TABLE_NAME
         Query      : SELECT * FROM [@{item().TABLE_SCHEMA}].[@{item().TABLE_NAME}]
       Sink:
         Dataset    : ds_adls_bronze_parquet
         schemaName : @item().TABLE_SCHEMA
         tableName  : @item().TABLE_NAME

  5. Publish All

  6. Debug run → Monitor tab → should show all Sales tables copied to bronze ✅
     Verify in Portal: sadataeng260524dev → bronze → Sales → [TableName]/

PHASE 10 — Synapse Serverless SQL Views
Stage   : 🗄️  Expose Gold Delta files as SQL views (Synapse serverless)
──────────────────────────────────────────────────────────────────────────────
Purpose : Expose the Gold Delta Lake files as SQL views in Synapse so
          Power BI can query them with standard SQL without a dedicated pool.

─── 10.1 Create Serverless Database ─────────────────────────────────────────

  1. Portal → synapse-dataeng-dev → Open Synapse Studio

  2. Left menu → Develop → + SQL script

  3. Run:
       IF NOT EXISTS (SELECT * FROM sys.databases WHERE name = 'gold_db')
           CREATE DATABASE gold_db;

─── 10.2 Grant Synapse Access to ADLS ───────────────────────────────────────

  Portal → sadataeng260524dev → Access Control (IAM) → Add role assignment
    Role     : Storage Blob Data Reader
    Assign to: Managed Identity → synapse-dataeng-dev

─── 10.3 Create Views for Each Gold Table ───────────────────────────────────

  In Synapse Studio → Develop → + SQL script → connect to gold_db

  Template for each table (repeat per Sales table):

    CREATE OR ALTER VIEW Sales_Customer AS
    SELECT *
    FROM OPENROWSET(
        BULK 'https://sadataeng260524dev.dfs.core.windows.net/gold/Sales/Customer/',
        FORMAT = 'DELTA'
    ) AS [result];

  Tables to create views for:
    - Customer
    - Product
    - ProductCategory
    - SalesOrderHeader
    - SalesOrderDetail
    - Address
    - CustomerAddress

─── 10.4 Create Stored Procedure (Dynamic — creates all views at once) ──────

    CREATE OR ALTER PROCEDURE sp_create_gold_views
    AS
    BEGIN
        DECLARE @sql NVARCHAR(MAX)
        DECLARE @tables TABLE (tbl NVARCHAR(100))

        INSERT INTO @tables VALUES
            ('Customer'), ('Product'), ('ProductCategory'),
            ('SalesOrderHeader'), ('SalesOrderDetail'),
            ('Address'), ('CustomerAddress')

        DECLARE @tbl NVARCHAR(100)
        DECLARE cur CURSOR FOR SELECT tbl FROM @tables
        OPEN cur
        FETCH NEXT FROM cur INTO @tbl
        WHILE @@FETCH_STATUS = 0
        BEGIN
            SET @sql = N'
            CREATE OR ALTER VIEW Sales_' + @tbl + N' AS
            SELECT * FROM OPENROWSET(
                BULK ''https://sadataeng260524dev.dfs.core.windows.net/gold/Sales/' + @tbl + N'/'',
                FORMAT = ''DELTA''
            ) AS [result]'
            EXEC sp_executesql @sql
            FETCH NEXT FROM cur INTO @tbl
        END
        CLOSE cur
        DEALLOCATE cur
    END

    -- Run it:
    EXEC sp_create_gold_views

  Verify:
    SELECT TOP 10 * FROM Sales_Customer;  -- should return rows ✅

─── 11.3 Gender Column (Derived) ────────────────────────────────────────────

  The Customer table has a TITLE column (Mr./Ms./Mrs./Miss/Dr.).
  Derive gender from it:

  Power BI → Transform data → Add column → Custom column:
    Column name: Gender
    Formula:
      if [TITLE] = "Mr." or [TITLE] = "Dr." then "Male"
      else if [TITLE] = "Ms." or [TITLE] = "Mrs." or [TITLE] = "Miss" then "Female"
      else "Unknown"

─── 11.4 Build Dashboard Visuals ────────────────────────────────────────────

  Visual 1 — Card: Total Products Sold
    Value: COUNT(Sales_SalesOrderDetail[ORDER_QTY])

  Visual 2 — Card: Total Revenue
    Value: SUM(Sales_SalesOrderDetail[LINE_TOTAL])
    Format: Currency

  Visual 3 — Donut Chart: Gender Split
    Legend : Sales_Customer[Gender]
    Values : COUNT(Sales_Customer[CUSTOMER_ID])

  Visual 4 — Bar Chart: Revenue by Product Category
    Axis  : Sales_ProductCategory[NAME]
    Values: SUM(Sales_SalesOrderDetail[LINE_TOTAL])

  Slicers:
    - Sales_SalesOrderHeader[ORDER_DATE]     (date range slicer)
    - Sales_ProductCategory[NAME]            (product category)
    - Sales_Customer[Gender]                 (gender)
